Итераторы (1)

Полуинтервалы в программировании — это концепция, используемая для определения диапазона значений, который включает один из своих конечных элементов и исключает другой. Это часто используется для определения диапазона индексов в массивах и других структурах данных. Например, в Python, если у нас есть список a, и мы хотим взять подсписок, начиная с индекса i и заканчивая индексом j, мы можем написать a[i:j]. Это включит элемент a[i], но исключит элемент a[j]. Таким образом, это определяет полуинтервал [i, j).

Итератор в программировании - это объект, который позволяет программистам перебирать элементы коллекции. Итераторы обычно определяют два основных метода: один для получения текущего элемента (часто называемый next) и один для перехода к следующему элементу. Они обеспечивают универсальный способ доступа к элементам коллекции, независимо от ее конкретной реализации.

Итераторы и указатели оба являются инструментами, используемыми в программировании для доступа и манипулирования элементами коллекции или массива. Однако между ними есть несколько ключевых различий. Указатель - это переменная, которая хранит адрес другой переменной. Указатели обычно используются в низкоуровневом программировании. Они могут быть использованы для доступа к памяти, манипулирования структурами и массивами данных.

С другой стороны, итератор представляет собой более высокоуровневый паттерн, который предоставляет универсальный интерфейс для перебора элементов коллекции, независимо от ее конкретной реализации. Итераторы обычно предоставляют методы для доступа к текущему элементу и перехода к следующему элементу. Они обеспечивают более безопасную и удобную альтернативу использованию непосредственных указателей для доступа к элементам коллекции.

#include <vector>

using namespace std;

template <typename T>

{
    for (auto it = it_begin; it != it_end; it++)
    {
        cout << *it << " ";
    }
}

int main()
{

    vector<string> vegtbls = {"cucumber", "potato", "cucumber"};
    vegtbls.push_back("tomato");
    vegtbls.resize(5);
    vegtbls[4] = "baclajan";

    vector<string>::iterator it = vegtbls.begin();
    vegtbls.insert(it, "luk");

    DisplayVector(vegtbls.begin(), vegtbls.end());

    return 0;
}

remove_if перекидывает элементы которые пободают под условия в конец контейнера. и возвращает итератор на новый желаемы конец вектора

#include <iostream>
#include <vector>

    for (auto it = it_begin; it != it_end; it++)
    {
        cout << *it << " ";
    }

int main()
{

    vector<string> vegtbls = {"cucumber", "potato", "cucumber", "kkkkk"};

    auto it = remove_if(vegtbls.begin(), vegtbls.end(), [](const string &s)
                        { return s[0] == 'k'; });

    vegtbls.erase(it, vegtbls.end());

    DisplayVector(vegtbls.begin(), vegtbls.end());

    return 0;
}

Итоги:

По документации итераторы находятся в библиотеке <iterator>.

4 категории итераторов:

  1. input итераторы(из которых можно читать)
  1. output итераторы (в которые можно записывать)
  1. forward(обычные итераторы с которых можно читать)
  1. random (к которым применима арифметика)

Input итераторы - это тип итераторов, из которых можно только читать. Они используются для однократного прохождения по коллекции в одном направлении.

Forward итераторы, с другой стороны, предоставляют более больший функционал. Они также позволяют читать данные, но в отличие от input итераторов, forward итераторы могут использоваться для многократного прохождения по коллекции. Это означает, что вы можете проходить по коллекции с использованием forward итератора столько раз, сколько вам нужно, в то время как с input итератором вы можете это сделать только один раз.
