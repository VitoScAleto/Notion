Ассемблер Основы (1)

Ассемблер GAS (GNU Assembler) - это ассемблер, созданный в рамках проекта GNU.

При компиляции g++ используется ассемблер GAS (GNU Assembler).

Из недостатков можно отметить, что GAS использует синтаксис, отличный от синтаксиса Intel (а именно синтаксис AT&T), что может немного усложнить перенос программ с одного ассемблера на другой. Хотя последние версии GCC включают параметр «-masm», который при значении "-masm=intel" позволяет встраивать код ассемблера с использованием синтаксиса Intel. Эквивалентным параметром для GAS является "-msyntax=intel" или использование директивы ".intel_syntax"

Глава 1. Введение

Для описания состояния сигнала информатике используется термина бит. По сути бит является наименьшей единицей информации в компьютере. Бит может иметь значение 1 (есть сигнал, что обычно соответствует напряжению от 2 до 5 V) или 0 (сигнал отсутствует или слабый - обычно от 0 до 2 V).

8 битов составляют байт. Фактически байт — это наименьшая единица информации, которую можно прочитать или записать в память большинством современных процессоров.

Основу программы на ассемблере составляют инструкции - некоторые действия, например, сложение двух значений, помещение в регистр значения и т.д. Каждой инструкции сопоставляется определенный машинный двоичный код, который также называется кодом инструкции или кодом операции (опкод, opcode).

Написание машинного кода вручную возможно, но излишне громоздко. На практике вместо опкодов применяются так называемые мнемоники - человекочитаемые названия инструкций.

Основные опкоды (инструкции) в ассемблере включают:

  • MOV: Копирование данных из одного места в другое
  • ADD: Сложение двух чисел
  • SUB: Вычитание одного числа из другого
  • MUL: Умножение двух чисел
  • DIV: Деление одного числа на другое
  • JMP: Безусловный переход к другой инструкции
  • JZ, JNZ, JE, JNE, JL, JLE, JG, JGE: Условные переходы
  • PUSH: Добавление элемента в стек
  • POP: Удаление элемента из стека
  • CALL: Вызов подпрограммы
  • RET: Возврат из подпрограммы

! Программа состоит из набора подобных инструкций. Процессор запускает
программы через цикл выборки-выполнения (fetch-execute cycle). Компьютер
считывает по одной инструкции за раз.
Для этого процессор обращается к специальному регистру - указателю
команд (или регистр IP), который также называется программным счетчиком
(или PC) и который хранит адрес инструкции для выполнения.
По сути, компьютер выполняет бесконечный цикл следующих операций:

  1. Считывает инструкцию с адреса памяти, указанного указателем инструкции - регистром IP/PC
  1. Декодирует инструкцию (т. е. выясняет, что означает инструкция)
  1. Перемещает указатель инструкций (регистр IP/PC) к следующей инструкции
  1. Выполняет указанную инструкцию

! Особенность апкода в GAS:

многие инструкции имеют вариации для работы с различными типами данных. Все эти вариации инструкций образуются по одной схеме: к названию инструкции прибавляется суффикс, который указывает на тип данных.

  • q: для 64-разрядных чисел
  • l: для 32-разрядных чисел
  • w: для 16-разрядных чисел
  • b: для 8-разрядных чисел

В конце апкода дописывается суффикс. был move → movq для копирования 64-разрядных числе и т.д

В ассемблере есть несколько основных типов индексов:

  1. Прямой или абсолютный индекс: Этот тип индексации используется, когда адрес операнда указывается непосредственно в инструкции. Пример: MOV AX, [1234H].
  1. Индексированный адрес: В этом типе индексации адрес операнда вычисляется путем добавления смещения к содержимому регистра индекса. Пример: MOV AX, [BX+SI].
  1. Относительный индекс: Этот тип индексации используется, когда адрес операнда вычисляется путем добавления смещения к счетчику команд (PC). Это обычно используется в командах перехода.
  1. Базовый индекс с относительным смещением: Этот тип индексации используется, когда адрес операнда вычисляется путем добавления смещения к содержимому регистра базы. Пример: MOV AX, [BP+8].
  1. Базовый индекс с индексированным смещением: Этот тип индексации используется, когда адрес операнда вычисляется путем добавления содержимого регистра индекса и смещения к содержимому регистра базы. Пример: MOV AX, [BP+SI+8].

В ассемблере используются различные типы регистров, каждый из которых предназначен для выполнения конкретных функций. Существуют следующие категории регистров:

  1. Основные регистры: Это регистры общего назначения, которые используются для выполнения арифметических и логических операций. Основными регистрами являются AX, BX, CX и DX.
  1. Регистры сегментов: Эти регистры используются для хранения сегментов кода, данных и стека. Основные регистры сегментов включают CS (регистр сегмента кода), DS (регистр сегмента данных), SS (регистр сегмента стека) и ES (регистр дополнительного сегмента).
  1. Регистры индекса: Эти регистры обычно используются для хранения смещений внутри сегментов. Они включают в себя SI (регистр исходного индекса) и DI (регистр индекса назначения).
  1. Указательный регистр: Это регистры, используемые для хранения указателей на место в памяти. Они включают в себя SP (указатель стека) и BP (базовый указатель).
  1. Регистр счетчика команд: Этот регистр, обычно обозначаемый как IP (указатель инструкций), содержит адрес следующей инструкции, которую следует выполнить процессором.
  1. Регистр флагов: Регистр флагов используется для хранения состояния процессора. Он содержит флаги, которые указывают на результат последней выполненной операции.

extern "C" {
    void func();
}

int main() {
    func();
    return 0;
}

asm(R"(
.func:
    mov eax, 0
    ret
)");
