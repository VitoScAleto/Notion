Существование нейтрального элемента: a * e = e * a = a
Существование обратного элемента: a * a^(-1) = a^(-1) * a = e

Кольцо - алгебра с двумя операциями (обычно сложение и
умножение), где сложение образует абелеву группу, а умножение
ассоциативно и дистрибутивно относительно сложения.

Абелева группа по сложению — это абелева группа, в которой операция
называется сложением. Она удовлетворяет следующим аксиомам:

Ассоциативность сложения: ((a + b) + c = a + (b + c)) для всех элементов (a, b,
c).
Коммутативность сложения: (a + b = b + a) для всех элементов (a, b).
Существование нейтрального элемента (нуль): существует элемент (0) такой,
что (a + 0 = 0 + a = a) для любого элемента (a).
Существование обратного элемента (отрицание): для каждого элемента (a)
существует элемент (-a) такой, что (a + (-a) = (-a) + a = 0).
Ассоциативность умножения: (a * b) * c = a * (b * c)
Дистрибутивность: a * (b + c) = (a * b) + (a * c)

  1. Ассоциативное кольцо - Кольцо, в котором операция умножения
    ассоциативна.
  1. Свойства операции умножения:
    • Ассоциативность умножения: (a * b) * c = a * (b * c).
  1. Распределительное свойство: Умножение распределяется относительно
    сложения:
    • a * (b + c) = (a * b) + (a * c).
    • (a + b) * c = (a * c) + (b * c).

Алиса генерирует свой секретный ключ a (допустим, 6) и вычисляет свой

Боб генерирует свой секретный ключ b (допустим, 9) и вычисляет свой

B = g^b mod p = 5^9 mod 23 = 1953125 mod 23 = 11.

Алиса и Боб обмениваются публичными ключами: Алиса отправляет свой
ключ A (8) Бобу, а Боб отправляет свой ключ B (11) Алисе.

Алиса вычисляет общий секретный ключ s:

s = B^a mod p = 11^6 mod 23 = 1771561 mod 23 = 9.

Боб вычисляет общий секретный ключ s:

s = A^b mod p = 8^9 mod 23 = 134217728 mod 23 = 9.

Параметры

  1. Выбирается большое простое число p.
  1. Выбирается первообразный корень g по модулю p.

Генерация ключей

  1. Вычисляется открытый ключ y как y = g^x mod p.
  1. Открытый ключ: (p, g, y).
  1. Приватный ключ: x.

Шифрование

Для шифрования сообщения m отправитель использует открытый ключ
получателя (p, g, y):

  1. Выбирается случайное число k, где 1 < k < p-1.
  1. Вычисляется c_1 = g^k mod p.
  1. Вычисляется c_2 = m * y^k mod p.
  1. Шифротекст: (c_1, c_2).

Расшифрование

Для расшифрования шифротекста (c_1, c_2) получатель использует свой
приватный ключ x:

  1. Вычисляется s = c_1^x mod p.
  1. Вычисляется s^(-1) (обратное значение s по модулю p).
  1. Расшифрованное сообщение: m = c_2 * s^(-1) mod p.

Пример

  1. Пусть p = 23, g = 5.
  1. Выбирается приватный ключ x = 6.
  1. Вычисляется открытый ключ y = g^x mod p = 5^6 mod 23 = 8.

Шифрование

  1. Сообщение m = 15.
  1. Выбирается случайное число k = 10.
  1. Вычисляется c_1 = g^k mod p = 5^10 mod 23 = 9.
  1. Вычисляется c_2 = m * y^k mod p = 15 * 8^10 mod 23 = 21.
  1. Шифротекст: (c_1, c_2) = (9, 21).

  1. Вычисляется s = c_1^x mod p = 9^6 mod 23 = 4.
  1. Вычисляется s^(-1) mod p. Обратное значение 4 по модулю 23: 4^(-1) ≡ 6

  1. Расшифрованное сообщение: m = c_2 * s^(-1) mod p = 21 * 6 mod 23 = 15.

Big O нотация — это математическая нотация, используемая для описания
асимптотического поведения функций. В информатике она часто
применяется для описания временной или пространственной сложности
алгоритмов, то есть как быстро растет время выполнения или объем памяти,
требуемый алгоритмом, по мере увеличения размера входных данных.

Основные функции Big O нотации:

  1. O(1) — Константная сложность: время выполнения не зависит от
    размера входных данных. Пример: доступ к элементу массива по индексу.
  1. O(log n) — Логарифмическая сложность: время выполнения растет
    логарифмически с увеличением размера входных данных. Пример: бинарный
    поиск.
  1. O(n) — Линейная сложность: время выполнения растет линейно с
    увеличением размера входных данных. Пример: простой проход по массиву.
  1. O(n log n) — Линейно-логарифмическая сложность: время
    выполнения растет быстрее, чем линейно, но медленнее, чем квадратично.
    Пример: алгоритмы сортировки слиянием и быстрой сортировки.
  1. O(n^2) — Квадратичная сложность: время выполнения растет
    квадратично с увеличением размера входных данных. Пример: сортировка
    пузырьком, сортировка вставками.
  1. O(2^n) — Экспоненциальная сложность: время выполнения растет
    экспоненциально с увеличением размера входных данных. Пример: решение
    задачи коммивояжера методом полного перебора.
  1. O(n!) — Факториальная сложность: время выполнения растет
    факториально с увеличением размера входных данных. Пример: полный
    перебор всех перестановок.

Big O нотация позволяет сравнивать эффективность алгоритмов, игнорируя
постоянные множители и менее значимые члены, фокусируясь на росте
функции при больших значениях входных данных.
