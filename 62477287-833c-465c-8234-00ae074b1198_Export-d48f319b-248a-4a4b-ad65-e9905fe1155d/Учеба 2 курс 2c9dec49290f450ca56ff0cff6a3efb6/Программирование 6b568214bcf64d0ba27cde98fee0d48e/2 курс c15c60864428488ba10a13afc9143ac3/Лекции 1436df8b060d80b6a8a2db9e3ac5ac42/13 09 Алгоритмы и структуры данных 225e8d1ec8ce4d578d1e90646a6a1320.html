13.09 Алгоритмы и структуры данных

Алгоритм - метод решенние определнной задачи, последовательное выполненеие операции над исходными данными с целью получения результата.

Алгорит характерезуется 3-мя показателями:

  • Конечность
  • Определенность
  • Эффективность(все шаги просты, время выполнения лежит в определенных границах)

Эмпирический анализ алгоритма(Время):

  • из-за разного железа неьзя сравнить одинаково

Математичесий анализ алгоритма:

  • оценить производительности программы
  • установить значение параметров алгоритмов
  • сравнить разные алгоритмы, предназначенные для решение одной задачи

При анализе алгоритма используется 3 оценки: O омега тетта

  1. O большое. Дано T, f: N→R+(есть две функции которые создают отображение натуральных чисел только на положительном участке).

    O ограничивает функцию сверху

Пример: 1/3n^2 - n = O(n^2)

  1. Тэта - это Обольшое и омега вмете

  1. Правило произведений
  1. O(cf(N)) = O(f(N))

    1.Время выполнения последовательности операторов определяется с помощью правила сумм.
    2. Время выполнения оператора присваивания О(1)
    3. Время выполнения условных операторов состоит из времени выполнения последовательности условно исполняемых операторов и времени вычисления логического выражения.
    4. Время выполнения оператора цикла является суммой времени всех исполняемых итераций цикла и времени вычисления условия прекращения цикла.

Структуры данных

Скалярный тип данных - (int, float and etc)

Структурированный тип данных - объединение всех типов.

Абстрактный тип данных - обобщение типов данных, которое характеризуется набором операций, выполняемых с данными(доступ, создание, добавление, удаление и тд)

Концепция абстрактного типа данных позволяет выделять выделять фундаментальные структуры(массив, структура, файл). Более сложные структуры могут быть динамическими(деревья, списки, очередь и тд).

Инкапсуляция - это процесс отделения друг от дурга элементов ообъекта, определяющих его устройство и поведение; инкапсуляция служит для того, чтобы изолировать контрактные обязательства абстракции от из реализации.

Интерфейс - отражает внешнее поведение абстракции, специфируя поведение всех объектов данного класса.

Внтуренняя реализация - описывает представление абстракции и механизмы достижения желаемого поведения объекта.

Для формирования АТД нужно задать множество операции, которые зависят от АТД

Линейный список посрелством массива.Последоватльное распределение памяти.

Связной список.Связное распредеение памяти, при котором к элементам осуществляется последовательный доступ

Связный список. Соглашения при организации

  1. Последний узел списка содержит нулевой указатель
  1. Последний узел списка содержит указатель на первый узел, что делает список циклическим.
  1. Последний узел списка содержит указатель на фиктивный узел (хвост списка).
  1. Указатель на список содержит фиктивный ведущий элемент (голова списка).

В зависимости от выбранной модели будет различаться реализация как инициализации списка, так и других операций.

Пример 1. Реализация фиктивных узлов оптимизирует задачу вставки и удаления единственного элемента списка.
